Web-kehitys kurssilla tehty ensimmäinen JavaScript Tehtävä

1. Koodi toimii siten, että käyttäjä syöttää sanan, joka  otetaan ja luodaan sen käänteinen versio. Tämä tehdään käymällä läpi sanan kaikki kirjaimet takaperin, niin että ensimmäinen kirjain tulee viimeiseksi ja viimeinen kirjain tulee ensimmäiseksi.
2. Kun sana on käännetty, verrataan alkuperäistä sanaa ja käännettyä sanaa. Jos ne ovat täsmälleen samat, se tarkoittaa,että sana on palindromi, eli se on sama, vaikka luettaisiin etuperin tai takaperin.
3. Jos sanat ovat samat, näytetään käyttäjälle viesti, joka kertoo, että sana on palindromi. Jos taas sanat eivät ole samat, näytetään viesti, joka kertoo, että sana ei ole palindromi.
Alunperin sovellus toimi vain komentokehoitepohjaisesti, mutta se on myös implementoitu nettisivulle, kuten alhaalta näkee.

Palindromi testeri

Html-ystävällinen javascript koodi.

Tämä koodi koostuu vain yhdestä funktiosta, jossa käyttäjän syöttämä sana haetaan HTML-sivulta ja tallennetaan muuttujaan. Sen jälkeen sana käydään läpi takaperin ja muodostetaan uusi merkkijono, joka sisältää sanan kirjaimet käänteisessä järjestyksessä. Lopuksi alkuperäistä ja käännettyä sanaa verrataan keskenään. Jos ne ovat täsmälleen samat, käyttäjälle näytetään vihreällä tekstillä viesti, että sana on palindromi. Muussa tapauksessa näytetään punainen viesti, joka kertoo, ettei sana ole palindromi.
function palindrome() {
              var sana = document.getElementById("wordInput").value;
            
              var sanaUpsideDown = "";
            
              for (let i = sana.length - 1; i >= 0; i--) {
                sanaUpsideDown += sana[i];
              }
            
              var resultElement = document.getElementById("result");
              if (sana === sanaUpsideDown) {
                resultElement.textContent = "Sana on palindromi.";
                resultElement.style.color = "green";
              } else {
                resultElement.textContent = "Sana ei ole palindromi.";
                resultElement.style.color = "red";
              }
            }

Alkuperäinen Komentokehoitepohjainen koodi

Tässä komentokehoitepohjaisessa koodissa eroavaisuutena on readline-sync-kirjaston käyttö, joka mahdollistaa käyttäjän syötteen lukemisen suoraan komentoriviltä ilman asynkronisia tapahtumia. Toisin kuin selainpohjaisessa versiossa, jossa syöte haetaan HTML-elementistä document.getElementById -metodilla, tässä syöte otetaan suoraan käyttäjältä komennolla input.question(). Lisäksi tulos tulostetaan suoraan komentokehoitteeseen console.log-komennolla, eikä näytetä HTML-sivulla värillisenä tekstinä.
function palindrome() {
              var input = require("readline-sync");
              var sana = input.question("Syötä jokin sana: ");
            
              var sanaUpsideDown = "";
            
              for (let i = sana.length - 1; i >= 0; i--) {
                sanaUpsideDown += sana[i];
              }
            
              if (sana == sanaUpsideDown) {
                console.log("Sana on palindromi.");
              } else {
                console.log("Sana Ei ole palindromi.");
              }
            }
            
            palindrome();

Komentokehoitepohjainen puhelinluettelo tehtävä

Tässä tehvässä on kyse komentokehoitepohjaisesta javascript ohjelmasta, jossa pyritään mallintamaan puhelinluetteloa. Puhelinluettelo toimii henkilö luokan kautta, jossa henkilö-oliolla on nimi ja puhelinnumero. Ohjelmassa on neljä eri toimintoa, joiden välillä navikoidaan switch-case pohjaisen käyttöliittymän avulla. Jokaisen toiminnon suorittamisen jälkeen palataan jälleen alkupisteeseen, jossa voi valita toisen toiminnon. Ensimmäinen toiminto mahdollistaa koko puhelinluettelon tulostamisen komentokehtoitteeseen. Toinen toiminto mahdollistaa uuden henkilön lisäämisen puhelinluetteloon. Kolmas toiminto mahdollistaa puhelinnumeron hakemisen nimen perusteella ja, jos henkilöä ei löydy siitä ilmoitetaan. ja neljäs mahdollistaa sovelluksen lopettamisen.

        function HaeHenkiloa(puhelinluettelo, nimi) {

          if (henkilo) {
            return henkilo.puhelinnumero; // Palautetaan puhelinnumero

        do {

            "Valitse toiminto (1: Näytä kaikki, 2: Lisää henkilö, 3: Hae henkilö, q: Lopeta): "

        } while (Valinta !== "q");

Getterit ja setteri 

Gettereitä ja settereitä käytetään, koska tarjoavat kontrolloidun tavan hakea ja asettaa olion ominaisuuksia. getter mahdollistaa ominaisuuden hakemisen erityisellä logiikalla, kuten syntymävuoden palauttamisen, ja setter mahdollistaa ominaisuuden asettamisen tiettyjen ehtojen mukaisesti. Jos niitä ei käytetä, ominaisuuksiin voidaan silti suoraan lukea ja kirjoittaa, mutta se voi aiheuttaa ongelmia tietyissä tilanteissa. Gettereihein ja settereihin ominaisuudet merkitään this_.jotain, this.jotain sijasta, koska näin voidaan välttää rekursiivinen looppi, joak voisi kaataa ohjelman lopulta.

                  return this._omapaino;
                }
              
                set omapaino(omapaino) {
                  this._omapaino = omapaino + " kg";
                }
              
                get laji() {
                  return this._laji;
                }
                set laji(laji) {
                  this._laji = laji;
                }

                }

                }
              
                get linkki() {
                  return this._linkki; // Getter palauttaa _linkki-attribuutin

                set linkki(linkki) {
                  if (!/^https?:\/\//.test(linkki)) {

                }

            async function postFormDataAsJson({ url, formData }) {
              const plainFormData = Object.fromEntries(formData.entries());
              const formDataJsonString = JSON.stringify(plainFormData);
              const fetchOptions = {
                method: "POST",
                headers: { "Content-Type": "application/json", Accept: "application/json" },
                body: formDataJsonString,
              };
              const response = await fetch(url, fetchOptions);
              if (!response.ok) {
                const errorMessage = await response.text();
                throw new Error(errorMessage);
              }
              return response.json();
            }

Olen myös tutustunut lomakkeen käsittelyyn ja asynkronisiin operaatioihin fetch-metodilla. Vielä opittavaa voisi olla syvempää ymmärrystä virheiden käsittelystä, tietoturvasta (esim. SQL-injektioiden ehkäisy, autentikointi) ja mahdollisesti paremmista sovellusrakenteista (esim. MVC-arkkitehtuuri, frontend-backend-erottelu).
